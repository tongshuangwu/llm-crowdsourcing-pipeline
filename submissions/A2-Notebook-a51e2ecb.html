Writeup 0: Task and Strategy Description¶

I'm replicating the pipelining workflow for doing Semantic Role Labeling(SRL) as originally described in the paper Question-Answer Drive Semantic Role Labeling: Using Natural Language to Annotate Natural Language. This paper proposes a dataset of semantic role labeling tasks reframed into QA tasks for annotators, obviating the need for long annotation manuals, such as the one for PropBank, which at the time of the QA-SRL paper's writing was 89 pages. It now seems to be significantly shorter, at 66 pages.

PropBank parsing, because of its relative popularity for the SRL task, will be the one-step prompt that we use to compare against the QA-SRL parse. The QA-SRL paper notes that the roles that it defines provide good coverage of the PropBank roleset, so these tasks should be roughly comparable. The reason we can't benchmark an apples-to-apples comparison of QA-SRL in one step to QA-SRL in multiple steps is because SRL as a task is relatively specialized, and would need an annotation guide, which the dataset is not well-structured for. Instead, we refer to PropBank, which is already in the training data for most language models. For our multi-step pipeline, we could equivalently prompt for individual PropBank roles, rather than QA-SRL ones, but QA-SRL is designed specifically for this purpose.

QA-SRL consists of two tasks, identifying the predicates in a sentence, followed by identifying the predicate's associated semantic roles. Here, we will focus on the latter task: given a predicate, can we identify what its corresponding semantic roles are? In the case of QA-SRL, these roles are defined in terms of templated questions, as seen in the examples.

  • Task Description: The task is to identify the occupants of semantic roles corresponding to a given predicate. Many different definitions of these semantic roles have been created; perhaps most common is PropBank. The crowdsourcing workflow proposed in the paper I'm relying on does not use PropBank; it instead defines semantic roles as the answers to a series of questions. In comparing the direct vs. pipelined approaches, we will use PropBank annotation for the direct requests with no specific pipelining; the pipelined approach will use the QA format. We rely on the fact that PropBank is common enough as a linguistic framework that GPT models have almost certainly been exposed to it in their training data, and in fact now generate cheeky responses about the subjectivity of linguistic annotation with an embarrassing number of hashtags.

def prompt_for_propbank_roles(sentence, predicate):
    prompt = f"""For the following sentence and predicate, provide a list of all the associated PropBank semantic roles. 

Sentence: {sentence}

Predicate: {predicate}

Propbank Roles:
"""

    completion = openai.Completion.create(prompt=prompt, **openai_config)
    output = completion["choices"][0]["text"]

    print(f"Input Sentence: {sentence}\n\nPredicate: {predicate}")
    print(f"Output:\n{output}")

    return completion

def prompt_for_qasrl(sentence_predicate_group):
    sentence = sentence_predicate_group.iloc[0]["sentence"]
    predicate = sentence_predicate_group.iloc[0]["predicate"]

    print(f"Input Sentence: {sentence}\nPredicate: {predicate}\n\n")

    for idx, row in sentence_predicate_group.iterrows():
        question = format_question(row["question"])
        prompt  = f"""For the following sentence and predicate, answer the question.
Sentence: {sentence}

Predicate: {predicate}

Question: {question}

Answer: 
"""
        completion = openai.Completion.create(
            prompt=prompt,
            **openai_config
        )
        output = completion["choices"][0]["text"]
        print(f"Question: {question}\nPredicted Answer:{output}\nActual Answer{row['answers']}\n\n")

Reflect on prompting effectiveness¶

Write some paragraphs to describe: Did you find the pipeline prompting workflow effective? Why or Why not?

I thought the pipelining prompting workflow was pretty effective - the answers that the model produced were generally correct. However, in further examples, such as the first sentence "On Friday, Clark...", with the predicate "post", the question "where doesn't someone post something" was answered with the same answer as "what doesn't someone post" - I think this might point at an inflexibility of LLMs that won't handle well questions that don't look "well-formed" even, if a human could deduce their meaning.

I do want to reiterate that this seemed like a bit of an unfair comparison. Unfortunately, there's not really a way to specify the task of SRL in one prompt without either a) writing a whole annotation manual, or b) referring to an existing standard of SRL, which is what we did. I think if we had had some scaffolding here, the model may have been more successful at extracting PropBank semantic roles.

Envision possible improvements¶

Write some paragraphs to describe: What are some possible ways to further improve the pipeline or prompt design?

I think the major weakness of both of these prompts was the challenge of extracting structured information out, especially for the pipeline models. I think the (relatively) easy fixes here would be for either some few-shot prompting, so that the model has examples of how to produce the output in a format that's amenable for extraction into a structured format, or for a more careful cloze-style setup, which is harder to do in this case. Were we to use a masked language model, I'd suggest masking out the wh- question in each sentence, and having the model produce a token (or tokens) to fill in that mask to answer the question.

In the case of the direct prompt, I think using the Propbank frames, extracting their associated roles, and asking questions specific to those roles would be a major improvement, though that mirrors pretty closely the setup in the pipelined design.
